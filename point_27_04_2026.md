## Contexte d'exécution

L'agent opencode s'exécute dans un conteneur Docker (opencode v1.14.24 sur ubuntu:24.04) qui **n'a pas accès** aux fichiers du projet sur l'hôte. Les volumes montés sont limités à :
- `/workspace/` (projet courant — vide actuellement, seulement Dockerfile + opencode.sh)
- `~/.config/opencode/`
- `~/.opencode/`
- `~/.gitconfig:ro`
- `~/.ssh/:ro`

Pour travailler sur Secretarius, il faudra soit :
1. Copier-coller le contenu des fichiers dans le chat
2. Lancer opencode depuis `~/Secretarius/` sur l'hôte
3. Monter explicitement le volume Secretarius dans le conteneur
